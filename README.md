# Лабораторна робота №3

## Тема
Використання кастомних жестів у React Native та стилізація інтерфейсу мобільного застосунку.

## Реалізований функціонал
- Інтеграція бібліотеки `react-native-gesture-handler` для обробки складних взаємодій користувача.
- Реалізація одинарного та подвійного кліку через `TapGestureHandler` із використанням властивості `waitFor` для уникнення конфліктів.
- Використання `LongPressGestureHandler` для розпізнавання тривалого натискання (3 секунди).
- Реалізація вільного переміщення об'єкта по екрану за допомогою `PanGestureHandler` та `Animated` API.
- Обробка швидких жестів змахування (свайпів вправо та вліво) через `FlingGestureHandler`.
- Налаштування масштабування об'єкта в реальному часі за допомогою `PinchGestureHandler`.
- Глобальне управління станом застосунку (загальний рахунок, статистика жестів, статус місій) за допомогою React Context API.
- Використання Stack Navigator для зручного переходу між ігровим екраном та екраном прогресу.
- Стилізація компонентів за допомогою `StyleSheet` із застосуванням тіней, заокруглень та сучасних кольорових рішень.
- Додано 9-те кастомне завдання згідно з вимогами до лабораторної роботи.

## Технічні особливості та пояснення коду (FAQ)
Під час розробки було прийнято кілька важливих архітектурних рішень для оптимізації роботи жестів та анімацій:

1. **Чому в анімаціях використовується `useNativeDriver: false`?**
   Анімації жестів, які впливають на розміри екрану та компонування (такі як `translationX`, `translationY` або `scale`), у деяких версіях `react-native-gesture-handler` не можуть бути повністю делеговані нативному потоку без конфліктів із JS-потоком. Тому для стабільної роботи масштабування та переміщення драйвер вимкнено.
2. **Як працює плавне масштабування (`Animated.multiply`)?**
   Для реалізації `PinchGestureHandler` використовується множення базового масштабу (`initialScale`) на поточний масштаб від жесту щипка (`currentPinch`). Це дозволяє об'єкту плавно змінювати розмір у реальному часі та "запам'ятовувати" свій новий розмір після завершення жесту.
3. **Для чого потрібен проп `waitFor={doubleTapNode}`?**
   Він вказує системі розпізнавання жестів зачекати перед тим, як зарахувати одинарний клік. Це гарантує, що якщо користувач робить подвійний клік, одинарний клік не спрацює одночасно з ним.

Висновки
Бібліотека react-native-gesture-handler забезпечує значно кращу продуктивність порівняно зі стандартними обробниками React Native, оскільки взаємодія з жестами відбувається безпосередньо в нативному потоці (UI thread).
Використання Animated.Value та Animated.event у поєднанні з жестами дозволяє створювати плавні візуальні трансформації об'єктів (переміщення по осях X та Y, масштабування) без блокування основного потоку JavaScript.
Для коректної роботи кількох жестів на одному об'єкті (наприклад, одинарний і подвійний тап) критично важливо використовувати посилання (useRef) та властивість waitFor, що дозволяє системі правильно пріоритезувати дії користувача.
React Context є ефективним архітектурним рішенням для управління спільним станом у невеликих застосунках, що дозволяє легко синхронізувати лічильники жестів на головному екрані зі списком завдань на іншому екрані без проп-дрилінгу (prop drilling).
